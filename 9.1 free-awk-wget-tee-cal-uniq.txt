1. free Command
The free command in Linux is used to check your system’s memory status. It provides a snapshot of how RAM and swap memory are being utilized.

Basic Command:
ps
ps -a
ps -ef      # Full formatted list
# Filter by user or command
ps -ef | grep -i "ec2-user"
ps -ef | grep -i "jenkins"

free
Example:
[ec2-user@ip-172-31-14-201 ~]$ free

              total        used        free      shared  buff/cache   available
Mem:         974196      337836      569668        5224      207528      636360
Swap:             0           0           0
It shows:

Total memory → How much RAM is installed on the system.
Used memory → Memory currently in use by running programs.
Free memory → Completely unused memory.
Buffer/cache → Memory used by the system to cache files for faster access (not really “used” memory).
Swap → Extra virtual memory used when RAM is full.

Common Variants of free:
1. free -h → Human-Readable Format
2. free -m → Shows Memory in MB (megabytes)
3. free -g → Shows Memory in GB (gigabytes)

Example:

[ec2-user@ip-172-31-14-201 ~]$ free -h
              total        used        free      shared  buff/cache   available
Mem:          951Mi       329Mi       556Mi       5.1Mi       202Mi       621Mi
Swap:            0B          0B          0B

[ec2-user@ip-172-31-14-201 ~]$ free -m
              total        used        free      shared  buff/cache   available
Mem:            951         329         556           5         202         621
Swap:             0           0           0

[ec2-user@ip-172-31-14-201 ~]$ free -g
              total        used        free      shared  buff/cache   available
Mem:              0           0           0           0           0           0
Swap:             0           0           0




2. awk Command
The awk command is a powerful text-processing tool in Linux. 
It processes text line-by-line and column-by-column, making it ideal for parsing structured text files like CSVs or logs.
Let’s walk through real-world awk examples using a sample file with the following structure:

Sample Data File: employees.txt
John    35    50000    john@example.com
Alice   42    62000    alice@example.com
Bob     28    40000    bob@example.com
Nina    50    75000    nina@example.com
Sam     38    46000    sam@example.com
Priya   29    52000    priya@example.com
boult   43    45000    boult@example.com
Fields:

$1 = Name
$2 = Age
$3 = Salary
$4 = Email

Example 1. Print All Columns (Full Row)
awk '{print}' employees.txt
Example:
[ec2-user@ip-172-31-14-201 ~]$ cat employees.txt
John  35  50000  john@example.com
Alice 42  62000  alice@example.com
Bob   28  40000  bob@example.com
Nina  50  75000  nina@example.com
Sam   38  46000  sam@example.com
Priya 29  52000  priya@example.com
boult 43  45000  boult@example.com

[ec2-user@ip-172-31-14-201 ~]$ awk '{print}' employees.txt
John  35  50000  john@example.com
Alice 42  62000  alice@example.com
Bob   28  40000  bob@example.com
Nina  50  75000  nina@example.com
Sam   38  46000  sam@example.com
Priya 29  52000  priya@example.com
boult 43  45000  boult@example.com

So this command simply prints every line in the file, just like the cat command would.
awk is a text-processing tool.
'{}' contains the action you want to perform for each line.
print (without specifying any field like $1, $2, etc.) prints the entire line.
employees.txt is the input file.

Another way:
awk '{print $0}' employees.txt

Example:

[ec2-user@ip-172-31-14-201 ~]$ awk '{print $0}' employees.txt
John  35  50000  john@example.com
Alice 42  62000  alice@example.com
Bob   28  40000  bob@example.com
Nina  50  75000  nina@example.com
Sam   38  46000  sam@example.com
Priya 29  52000  priya@example.com
boult 43  45000  boult@example.com

{print $0}: $0 represents the entire current line in AWK.
This command prints the whole line just like awk '{print}'.

Example 2. Using Pattern Matching (Simpler Way)
awk '/Bob/' employees.txt
Example:
[ec2-user@ip-172-31-14-201 ~]$ awk '/Bob/' employees.txt
Bob  28  40000  bob@example.com
Searches for lines containing the word “Bob” (case-sensitive) in the file employees.txt and prints the entire line by default.
How it works:
/Bob/ is a pattern — AWK looks for any line that matches it.
If a line contains "Bob" anywhere, that line will be printed.

Example 3. Print Only Name and Salary
awk '{print $1, $3}' employees.txt
Example:
[ec2-user@ip-172-31-14-201 ~]$ awk '{print $1, $3}' employees.txt
John 50000
Alice 62000
Bob 40000
Nina 75000
Sam 46000
Priya 52000
boult 45000

$1 → Name (1st column)
$3 → Salary (3rd column)
So this command will print only the Name and Salary for each row in the file employees.txt.

Example 4. Flexible Way (Print First and Last Column Dynamically)
awk '{print $1, $NF}' employees.txt

Example:
[ec2-user@ip-172-31-14-201 ~]$ awk '{print $1, $NF}' employees.txt
John john@example.com
Alice alice@example.com
Bob bob@example.com
Nina nina@example.com
Sam sam@example.com
Priya priya@example.com
boult boult@example.com

$1 → First column
$NF → Last column (Number of Fields)
This command prints the first and last column of each row, no matter how many columns the file has.

Another example:
awk '{print $NF}' employees.txt

Example:

john@example.com
alice@example.com
bob@example.com
nina@example.com
sam@example.com
priya@example.com
boult@example.com

Example 5. Reverse Order (Print Email, Salary, Age, Name)
You can choose what to print in any order using awk. Decide the order you want to print them in.
awk '{print $4, $3, $2, $1}' employees.txt
Example:
[ec2-user@ip-172-31-14-201 ~]$ awk '{print $4, $3, $2, $1}' employees.txt
john@example.com 50000 35 John
alice@example.com 62000 42 Alice
bob@example.com 40000 28 Bob
nina@example.com 75000 50 Nina
sam@example.com 46000 38 Sam
priya@example.com 52000 29 Priya
boult@example.com 45000 43 boult

This command prints each line of the file, but in reverse order of the columns:
Email → $4
Salary → $3
Age → $2


Example 6. Show Line Numbers with Each Row (NR)
What is NR in awk?
NR stands for Number of Record.
It keeps track of the current line number being processed from the input file or stream.
Each time awk reads a new line, NR increments by 1.
awk '{print NR, $1,$2,$3,$4}' employees.txt
awk '{print NR, $0}' employees.txt

Example:
[ec2-user@ip-172-31-14-201 ~]$ awk '{print NR, $1,$2,$3,$4}' employees.txt
1 John 35 50000 john@example.com
2 Alice 42 62000 alice@example.com
3 Bob 28 40000 bob@example.com
4 Nina 50 75000 nina@example.com
5 Sam 38 46000 sam@example.com
6 Priya 29 52000 priya@example.com
7 boult 43 45000 boult@example.com

awk '{print NR, $1,$2,$3,$4}' employees.txt manually prints each field with line number

Example:

[ec2-user@ip-172-31-14-201 ~]$ awk '{print NR, $0}' employees.txt
1 John 35 50000 john@example.com
2 Alice 42 62000 alice@example.com
3 Bob 28 40000 bob@example.com
4 Nina 50 75000 nina@example.com
5 Sam 38 46000 sam@example.com
6 Priya 29 52000 priya@example.com
7 boult 43 45000 boult@example.com

Automatically prints line number + entire line
Cleaner and more flexible especially for files with unknown number of columns
Use:
awk '{print NR, $0}'
when you want a neat, complete output with line numbers.


Example 7. Display Only Rows Where Age < 40
awk '$2 < 40 {print }' employees.txt
Example:
[ec2-user@ip-172-31-14-201 ~]$ awk '$2 < 40 {print }' employees.txt
John 35 50000 john@example.com
Bob 28 40000 bob@example.com
Sam 38 46000 sam@example.com
Priya 29 52000 priya@example.com

$2 < 40 This condition checks if the second column (which might be Age) is less than 40.
{print} If the condition is true, it prints the entire line by default.
So this command prints all rows from employees.txt where the value in the second column is less than 40.


Example 8. Display Only Rows Where Age ≥ 40
awk '$2 >= 40 {print $0}' employees.txt
Example:

[ec2-user@ip-172-31-14-201 ~]$ awk '$2 >= 40 {print $0}' employees.txt
Alice 42 62000 alice@example.com
Nina 50 75000 nina@example.com
boult 43 45000 boult@example.com

$2 ≥ 40 This condition checks if the second column (usually Age) is greater than or equal to 40.
{print $0} If true, it prints the entire line.
So, this command prints all rows where the age is 40 or older from employees.txt.

Example 9. Display Records with Age Between 30 and 35
awk '$2 >= 30 && $2 <= 40 {print NR, $0}' employees.txt

Example:

[ec2-user@ip-172-31-14-201 ~]$ awk '$2 >= 30 && $2 <= 40 {print NR, $0}' employees.txt
1 John 35 50000 john@example.com
5 Sam 38 46000 sam@example.com

$2 ≥ 30 && $2 ≤ 40 This condition checks if the second column (Age) is between 30 and 40, inclusive.
{print NR, $0} If true, it prints:
NR → The current line number in the file
$0 → The entire line

Example 10. Display Rows Where Salary > 50000
awk '$3 > 50000' employees.txt

Example:

[ec2-user@ip-172-31-14-201 ~]$ awk '$3 > 50000' employees.txt
Alice 42 62000 alice@example.com
Nina 50 75000 nina@example.com
Priya 29 52000 priya@example.com

$3 refers to the third column — in your case, it's likely the Salary.
> 50000 filters the rows where salary is greater than 50,000.
Since no {print ...} block is given, awk prints the entire line by default.

Example 11. Combine Conditions: Age < 40 AND Salary > 45000
awk '$2 < 40 && $3 > 45000' employees.txt

Example:

[ec2-user@ip-172-31-14-201 ~]$ awk '$2 < 40 && $3 > 45000' employees.txt
John 35 50000 john@example.com
Sam 38 46000 sam@example.com
Priya 29 52000 priya@example.com

$2 < 40 — selects rows where the Age (2nd column) is less than 40
$3 > 45000 — AND the Salary (3rd column) is greater than 45000

The command prints only those lines from employees.txt that meet both conditions.
If you don’t explicitly specify a print statement inside {}, awk automatically prints the entire line that matches the condition.
Since there’s no { print } block, awk defaults to printing the whole line ($0) for every line where the condition is true.

Example 12. Highlight Specific Fields with Custom Output
awk '{print "Name: "$1", Age: "$2", Email: "$4}' employees.txt
Example:
[ec2-user@ip-172-31-14-201 ~]$ awk '{print "Name: "$1", Age: "$2", Email: "$4}' employees.txt
Name: John, Age: 35, Email: john@example.com
Name: Alice, Age: 42, Email: alice@example.com
Name: Bob, Age: 28, Email: bob@example.com
Name: Nina, Age: 50, Email: nina@example.com
Name: Sam, Age: 38, Email: sam@example.com
Name: Priya, Age: 29, Email: priya@example.com
Name: boult, Age: 43, Email: boult@example.com

$1, $2, $4 refer to the 1st (Name), 2nd (Age), and 4th (Email) columns.

The command prints each record with custom formatting using custom labels.
Example 13. Display Name and Email from Specific Line Range (Line 3 to 5)
awk 'NR>=3, NR<=5 {print "===> Name:", $1, "===> Email:", $4}' employees.txt
Example:
Name: Bob ===> Email: bob@example.com
Name: Nina ===> Email: nina@example.com
Name: Sam ===> Email: sam@example.com
NR >= 3, NR <= 5
This specifies a range of lines — from line 3 to line 5 (inclusive).
$1 and $4
$1 is the Name (1st column)
$4 is the Email (4th column)
Custom labels
The output will show Name and Email with clear labels and arrows ===>

4. tee – Write Output to File and Display
The tee command takes whatever output you get from a command and shows it on your screen and saves it to a file at the same time. 
This is handy when you want to keep a copy of what's happening but still watch it live.
Running a Script and Saving Logs
script.sh | tee output.txt
Example script:
[ec2-user@ip-172-31-14-201 ~]$ cat script.sh
pwd
echo "hello"
uptime
pwd
Run the script:
[ec2-user@ip-172-31-14-201 ~]$ sh script.sh | tee output.txt
/home/ec2-user
hello
18:23:52 up 4:52, 2 users, load average: 0.00, 0.00, 0.00
/home/ec2-user

View saved output:
[ec2-user@ip-172-31-14-201 ~]$ cat output.txt
/home/ec2-user
hello
18:23:52 up 4:52, 2 users, load average: 0.00, 0.00, 0.00
/home/ec2-user
The script’s output is both displayed on your terminal and saved into the file output.txt at the same time.
This means you can watch what's happening live and also keep a complete log for later review or sharing.

6. uniq – Remove Duplicate Lines
uniq removes consecutive duplicate lines in a file or input.
Note: uniq only removes consecutive duplicates. To work effectively, use sort first.
Example file:

[ec2-user@ip-172-31-0-204 ~]$ cat kknames.txt
raghu
sai
suman
suman
krishna
prasanth
sunitha
ram
jaga
suman
sai

Run uniq:
[ec2-user@ip-172-31-0-204 ~]$ uniq kknames.txt
raghu
sai
suman
krishna
prasanth
sunitha
ram
jaga
suman
sai
Only removes consecutive duplicates.
Example
If suman appears multiple times but not consecutively, all instances are shown.
If suman is repeated consecutively, only the first instance is shown.
Sort first:
[ec2-user@ip-172-31-0-204 ~]$ sort kknames.txt
jaga
krishna
prasanth
raghu
ram
sai
sai
suman
suman
suman
sunitha
Remove duplicates:
[ec2-user@ip-172-31-0-204 ~]$ sort kknames.txt | uniq
jaga
krishna
prasanth
raghu
ram
sai
suman
sunitha
First sort arranges all names alphabetically (grouping duplicates).
Then uniq removes duplicates (since now they are consecutive due to sorting).
This gives correct result: a unique list of all names in alphabetical order.







| Command  | Main Purpose            | Type of Operation            | Typical Use                    |
| -------- | ----------------------- | ---------------------------- | ------------------------------ |
| **grep** | Search text             | Pattern matching             | Find lines containing a word   |
| **awk**  | Process structured text | Field processing & scripting | Work with columns              |
| **sed**  | Edit text stream        | Stream editing               | Replace, delete, modify text   |
| **cut**  | Extract columns         | Column selection             | Extract specific fields        |
| **tr**   | Translate characters    | Character replacement        | Change case, remove characters |
| **sort** | Sort lines              | Ordering                     | Alphabetical / numeric sorting |
| **uniq** | Remove duplicates       | Duplicate filtering          | Remove repeated lines          |



grep – Search for Pattern

Used to search lines containing a pattern.

Example file:

apple
banana
orange
apple juice

Command:

grep apple file.txt

Output

apple
apple juice

Common options

grep -i word file      # ignore case
grep -v word file      # exclude word
grep -r word dir      # recursive search
grep -n word file     # show line number

Example (DevOps)

grep ERROR app.log

Find error logs.

2️⃣ awk – Column Based Processing

Used for working with structured text (columns).

Example file

John 35 50000
Alice 42 62000
Bob 28 40000

Print column 1

awk '{print $1}' file.txt

Output

John
Alice
Bob

Print salary > 50000

awk '$3 > 50000' file.txt

Key concept

$1 → column 1
$2 → column 2
$3 → column 3
NR → line number

Example

awk '{print NR,$0}'

Print line number + entire line

3️⃣ sed – Stream Editor

Used to modify text.

Example

hello world

Replace word

sed 's/world/linux/' file.txt

Output

hello linux

Common uses

sed 's/a/b/' file      # replace first match
sed 's/a/b/g' file    # replace all
sed '3d' file         # delete line 3
sed -n '2p' file      # print line 2

Example

sed 's/ERROR/WARNING/g' logs.txt

Replace error messages.

4️⃣ cut – Extract Columns

Used to extract specific fields.

Example file

John,35,50000
Alice,42,62000
Bob,28,40000

Extract name

cut -d',' -f1 file.txt

Output

John
Alice
Bob

Explanation

-d → delimiter
-f → field

Example

cut -d':' -f1 /etc/passwd

Print usernames.

5️⃣ tr – Translate Characters

Used for character replacement or deletion.

Example

hello world

Convert to uppercase

echo hello | tr a-z A-Z

Output

HELLO

Remove spaces

echo "hello world" | tr -d ' '

Output

helloworld

Example

tr ':' '\n'

Convert colon to new line.

6️⃣ sort – Sort Lines

Used to sort lines.

Example

orange
apple
banana

Command

sort file.txt

Output

apple
banana
orange

Useful options

sort -r      # reverse
sort -n      # numeric
sort -k2     # sort by column

Example

sort -n salaries.txt

Sort numbers.

7️⃣ uniq – Remove Duplicate Lines

Used to remove consecutive duplicate lines.

Example

apple
apple
banana
banana
orange

Command

uniq file.txt

Output

apple
banana
orange

But if duplicates are not consecutive, use sort.

sort file.txt | uniq

Useful options

uniq -c   # count duplicates
uniq -d   # show duplicates
uniq -u   # show unique lines

Example

sort log.txt | uniq -c

Count repeated log lines.

Quick Memory Trick
Command	Remember Like

cut	Cut columns

sort	Arrange lines

Real DevOps Example

Find unique error messages

grep ERROR app.log | sort | uniq -c
